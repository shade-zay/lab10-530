Lab 10 – My experience with Materialize CSS

Materialize CSS Framework Page

This page was created for Lab 10 to experiment with a web development framework. I chose Materialize CSS, a front-end library based on Google’s Material Design.

How I Installed Materialize

I used the CDN version of Materialize so I did not have to install anything locally:

  • Went to the Materialize website and opened the “Getting Started” section.
  • Copied the CSS CDN link into the <head> of this page.
  • Copied the JavaScript CDN link before the closing </body> tag.
  • Saved the file as a normal .html page and opened it in my browser to test.
How I Built This Page

After connecting Materialize, I used some of its basic components:

  • A navbar at the top using the nav and nav-wrapper classes.
  • A simple grid layout with the row and col s12 m6 classes.
  • Cards to separate each section of the explanation.
  • Chips at the bottom to highlight the tools used.
Difficulties and How I Solved Them

At first, I struggled a bit with the grid system and the way Materialize names its classes. I am more used to plain CSS, so remembering to wrap content inside a row and use col s12 m6 for columns took a few tries.

I also had to test a few different colour combinations to make sure the text stayed readable on the blue background. Opening the browser’s developer tools helped me quickly tweak the styles until everything looked balanced.

Finally, I tested the page on a smaller window to confirm that the layout was responsive. Materialize handled most of that automatically as long as I followed the grid structure.
